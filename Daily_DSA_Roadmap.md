# Daily DSA Roadmap — Concept + Practice + Revision

## How to Use This Roadmap

For every topic, follow this 3-step loop:
1. **Concept** — watch/read theory, understand *why* it works, not just *how* to code it
2. **Practice** — solve problems (easy → medium → hard) same day
3. **Revise** — revisit solved problems after 3 days and again after 10 days (spaced repetition)

Since you already know basics, do a **quick revision pass** on early topics (1–2 days each) and spend real time on topics where you're weak.

## Ongoing Daily Habit (Throughout All Weeks)

- **1 new problem + 1 revision problem, every single day** — don't skip even on light days
- **Maintain a "Patterns Notebook"** — write down the pattern/trick for each problem type (e.g., "two pointers for sorted array pair sum"), not the full code
- **After solving, always check the optimal solution** even if yours worked — compare time/space complexity
- **Track time taken per problem** — aim to reduce time as patterns become familiar

## Since You Already Know Some DSA

Don't restart from zero. In Week 1–2, spend just **1 day per already-known topic** doing 5–8 hard problems to confirm mastery, then move fast to your weak areas (usually Trees, Graphs, and DP are where most people need real time). Use this roadmap as a checklist — skip revision-only days for topics you're already confident in, but never skip Trees/Graphs/DP depth.
